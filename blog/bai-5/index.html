Bài 5 - Đa luồng (Multithreading) trong lập trình mạng

Đăng ngày 20/10/2025 lúc 00:00

1. Giới thiệu về đa luồng trong lập trình mạng

Trong lập trình mạng, đa luồng (Multithreading) cho phép chương trình xử lý nhiều kết nối cùng lúc, giúp tăng hiệu năng và độ phản hồi của server.
Một server đơn luồng (single-threaded) chỉ có thể phục vụ một client tại một thời điểm, trong khi đa luồng cho phép phục vụ nhiều client song song.

Ví dụ:

  • Khi nhiều người truy cập cùng lúc, mỗi kết nối client được xử lý bởi một luồng riêng biệt (thread).
  • Nhờ đó, các yêu cầu có thể được thực hiện đồng thời mà không bị chặn (blocking).

2. Khái niệm về Thread

Thread (luồng) là đơn vị nhỏ nhất của quá trình xử lý trong một chương trình.
Một chương trình có thể có nhiều luồng chạy song song (concurrent).

Cấu trúc cơ bản của luồng trong Java

class MyThread extends Thread {
    public void run() {
        System.out.println("Luồng đang chạy: " + Thread.currentThread().getName());
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        t1.start();
        t2.start();
    }
}

Hoặc sử dụng Runnable:

class MyTask implements Runnable {
    public void run() {
        System.out.println("Thực thi trong luồng: " + Thread.currentThread().getName());
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t1 = new Thread(new MyTask());
        Thread t2 = new Thread(new MyTask());
        t1.start();
        t2.start();
    }
}

3. Tại sao cần đa luồng trong lập trình mạng?

Vấn đề Giải pháp
Server bị treo khi 1 client gửi yêu cầu chậm Dùng thread riêng cho mỗi client
Cần xử lý nhiều kết nối đồng thời Tạo nhiều thread chạy song song
Giảm độ trễ phản hồi Các luồng hoạt động độc lập
Tận dụng CPU đa nhân Phân luồng giúp tận dụng hiệu năng tốt hơn

4. Mô hình Server đa luồng (Multithreaded Server)

Quy trình hoạt động

  1. ServerSocket lắng nghe kết nối đến.
  2. Mỗi khi có client kết nối, server sẽ:
    • Chấp nhận kết nối (accept())
    • Tạo một luồng mới để xử lý client đó.
  3. Luồng chính (main thread) quay lại chờ kết nối tiếp theo.

5. Ví dụ: Server đa luồng với TCP

File: MultiThreadedServer.java

import java.io.*;
import java.net.*;

class ClientHandler extends Thread {
    private Socket socket;

    public ClientHandler(Socket socket) {
        this.socket = socket;
    }

    @Override
    public void run() {
        try {
            BufferedReader input = new BufferedReader(new InputStreamReader(socket.getInputStream()));
            PrintWriter output = new PrintWriter(socket.getOutputStream(), true);

            String message;
            while ((message = input.readLine()) != null) {
                System.out.println("Nhận từ client " + socket.getPort() + ": " + message);
                output.println("Server nhận: " + message);
            }

            socket.close();
        } catch (IOException e) {
            System.out.println("Lỗi client " + socket.getPort() + ": " + e.getMessage());
        }
    }
}

public class MultiThreadedServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(5000);
        System.out.println("Server đa luồng đang lắng nghe...");

        while (true) {
            Socket socket = serverSocket.accept();
            System.out.println("Client mới kết nối: " + socket.getInetAddress());
            new ClientHandler(socket).start();
        }
    }
}

File: MultiClient.java

import java.io.*;
import java.net.*;

public class MultiClient {
    public static void main(String[] args) throws IOException {
        Socket socket = new Socket("localhost", 5000);
        BufferedReader input = new BufferedReader(new InputStreamReader(System.in));
        PrintWriter output = new PrintWriter(socket.getOutputStream(), true);
        BufferedReader serverResponse = new BufferedReader(new InputStreamReader(socket.getInputStream()));

        System.out.println("Kết nối đến server thành công!");
        String message;

        while (true) {
            System.out.print("Nhập tin nhắn: ");
            message = input.readLine();
            if (message.equalsIgnoreCase("exit")) break;
            output.println(message);
            System.out.println("Phản hồi: " + serverResponse.readLine());
        }

        socket.close();
    }
}

Cách chạy:

  1. Biên dịch: 
    javac MultiThreadedServer.java MultiClient.java
  2. Chạy server: 
    java MultiThreadedServer
  3. Chạy nhiều client cùng lúc (mỗi terminal 1 client): 
    java MultiClient

6. Ưu điểm và nhược điểm của đa luồng

Ưu điểm Mô tả
Xử lý đồng thời nhiều kết nối Mỗi client được phục vụ riêng biệt
Nhanh và hiệu quả Không cần chờ từng client hoàn tất
Tận dụng tài nguyên CPU Đặc biệt hữu ích trên máy nhiều nhân
Cải thiện trải nghiệm người dùng Phản hồi tức thì, không bị treo
Nhược điểm Mô tả
Tốn tài nguyên bộ nhớ Mỗi luồng cần stack riêng
Khó quản lý đồng bộ dữ liệu Cần cẩn trọng khi chia sẻ biến giữa các luồng
Có thể bị tràn luồng (thread explosion) Khi có quá nhiều kết nối cùng lúc

7. Cải tiến với Thread Pool

Thay vì tạo mới một luồng cho mỗi client, ta có thể dùng Thread Pool (bộ quản lý luồng):

  • Giới hạn số luồng tối đa đang chạy.
  • Tái sử dụng luồng sau khi client ngắt.
  • Tiết kiệm tài nguyên và kiểm soát tải tốt hơn.

Ví dụ với ThreadPoolExecutor:

import java.io.*;
import java.net.*;
import java.util.concurrent.*;

class Worker implements Runnable {
    private Socket socket;

    public Worker(Socket socket) {
        this.socket = socket;
    }

    public void run() {
        try {
            BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
            PrintWriter out = new PrintWriter(socket.getOutputStream(), true);
            String msg;
            while ((msg = in.readLine()) != null) {
                out.println("Server: " + msg);
            }
            socket.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

public class ThreadPoolServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(5000);
        ExecutorService pool = Executors.newFixedThreadPool(10);
        System.out.println("Server với Thread Pool đang hoạt động...");

        while (true) {
            Socket socket = serverSocket.accept();
            pool.execute(new Worker(socket));
        }
    }
}

8. Kết luận

Đa luồng là thành phần quan trọng trong lập trình mạng giúp xây dựng server hiệu quả và mở rộng dễ dàng.
Việc hiểu rõ cách hoạt động của luồng, đồng bộ hóa dữ liệu và tối ưu tài nguyên sẽ giúp lập trình viên triển khai các ứng dụng mạng mạnh mẽ, ổn định và hiệu suất cao.
