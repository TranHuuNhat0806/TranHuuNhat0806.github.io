Bài 6 - Lập trình Client – Server mô hình nhiều tầng (Multi-tier)

Đăng ngày 20/10/2025 lúc 00:00

1. Giới thiệu về mô hình nhiều tầng (Multi-tier)

Mô hình Client-Server nhiều tầng (Multi-tier Architecture) là một cách tổ chức hệ thống phần mềm sao cho việc xử lý dữ liệu, giao diện người dùng và truy cập cơ sở dữ liệu được tách biệt thành nhiều tầng độc lập.
Điều này giúp dễ bảo trì, mở rộng, và nâng cấp hệ thống mà không ảnh hưởng đến toàn bộ chương trình.

Các tầng phổ biến:

  1. Presentation Layer (Tầng giao diện người dùng)

    • Hiển thị dữ liệu và nhận thao tác từ người dùng.
    • Ví dụ: giao diện web, ứng dụng desktop, ứng dụng di động.

  2. Application Layer (Tầng xử lý nghiệp vụ / Logic)

    • Xử lý yêu cầu từ client, thực hiện logic nghiệp vụ.
    • Đóng vai trò trung gian giữa client và cơ sở dữ liệu.

  3. Data Layer (Tầng dữ liệu)

    • Lưu trữ, truy vấn và cập nhật dữ liệu trong cơ sở dữ liệu (Database).

Mô hình thường gặp:

  • 2-Tier (Client-Server)
  • 3-Tier (Client - Application Server - Database Server)

2. So sánh mô hình 2 tầng và 3 tầng

Tiêu chí 2-Tier 3-Tier
Số tầng 2 (Client – Server) 3 (Client – Application – Database)
Xử lý nghiệp vụ Trên Client hoặc Server Tách riêng ở Application Server
Bảo trì Khó hơn do phụ thuộc Dễ hơn do tách biệt logic
Bảo mật Trung bình Cao hơn
Mở rộng Hạn chế Linh hoạt, dễ mở rộng

3. Cơ chế hoạt động

  1. Client gửi yêu cầu (request) → tầng Application.
  2. Application Server xử lý logic nghiệp vụ, tương tác với Database Server nếu cần.
  3. Database Server truy xuất dữ liệu và trả kết quả về cho Application Server.
  4. Application Server xử lý và gửi phản hồi (response) lại cho Client.

Sơ đồ khái quát:

[ Client ] ⇄ [ Application Server ] ⇄ [ Database Server ]

4. Ưu điểm và nhược điểm của mô hình nhiều tầng

Ưu điểm:

  • Dễ quản lý và bảo trì vì các tầng độc lập.
  • Bảo mật tốt hơn: Client không truy cập trực tiếp cơ sở dữ liệu.
  • Tái sử dụng logic nghiệp vụ cho nhiều client khác nhau (web, mobile, desktop).
  • Dễ mở rộng hệ thống (scalable).

Nhược điểm:

  • Cấu trúc phức tạp hơn, tốn chi phí triển khai.
  • Giao tiếp giữa các tầng có thể tăng độ trễ (latency).
  • Cần thiết lập các giao thức truyền dữ liệu giữa các tầng (REST, gRPC, RMI…).

5. Ví dụ minh họa (Java)

a. Tầng Database – Database.java

import java.util.HashMap;
import java.util.Map;

public class Database {
    private static final Map<String, String> users = new HashMap<>();

    static {
        users.put("admin", "1234");
        users.put("user", "abcd");
    }

    public static boolean validateUser(String username, String password) {
        return users.containsKey(username) && users.get(username).equals(password);
    }
}

b. Tầng Application Server – AuthService.java

import java.io.*;
import java.net.*;

public class AuthService {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(6000);
        System.out.println("Application Server đang chạy...");

        while (true) {
            Socket client = serverSocket.accept();
            new Thread(() -> handleClient(client)).start();
        }
    }

    private static void handleClient(Socket socket) {
        try (
            BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
            PrintWriter out = new PrintWriter(socket.getOutputStream(), true)
        ) {
            String username = in.readLine();
            String password = in.readLine();

            boolean valid = Database.validateUser(username, password);
            if (valid) out.println("Đăng nhập thành công!");
            else out.println("Sai tài khoản hoặc mật khẩu!");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

c. Tầng Client – ClientApp.java

import java.io.*;
import java.net.*;

public class ClientApp {
    public static void main(String[] args) throws IOException {
        Socket socket = new Socket("localhost", 6000);
        System.out.println("Kết nối tới Application Server...");

        BufferedReader input = new BufferedReader(new InputStreamReader(System.in));
        PrintWriter out = new PrintWriter(socket.getOutputStream(), true);
        BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));

        System.out.print("Nhập tài khoản: ");
        String username = input.readLine();
        System.out.print("Nhập mật khẩu: ");
        String password = input.readLine();

        out.println(username);
        out.println(password);

        String response = in.readLine();
        System.out.println("Phản hồi: " + response);

        socket.close();
    }
}

Cách chạy:

  1. Biên dịch tất cả: 
    javac Database.java AuthService.java ClientApp.java
  2. Chạy AuthService trước: 
    java AuthService
  3. Sau đó chạy ClientApp: 
    java ClientApp

6. Ứng dụng thực tế

  • Web Application: Giao diện người dùng (frontend) kết nối với API (application layer) và cơ sở dữ liệu.
  • Game Online: Client gửi hành động, server xử lý logic, database lưu tiến trình.
  • Hệ thống ngân hàng / bán hàng / đặt vé: logic nghiệp vụ phức tạp cần tách thành nhiều tầng.

7. Kết luận

Mô hình Client–Server nhiều tầng (Multi-tier) giúp hệ thống linh hoạt, bảo mật và dễ mở rộng.
Việc hiểu rõ cách tổ chức và giao tiếp giữa các tầng là nền tảng quan trọng để phát triển các ứng dụng mạng quy mô lớn trong thực tế.
