Bài 8 - Bảo mật mạng trong lập trình (SSL/TLS, mã hóa)

Đăng ngày 21/10/2025 lúc 00:00

1. Giới thiệu

Trong lập trình mạng, bảo mật là yếu tố then chốt giúp đảm bảo:

  • Tính bí mật (Confidentiality): Dữ liệu không bị đọc lén.
  • Tính toàn vẹn (Integrity): Dữ liệu không bị thay đổi.
  • Xác thực (Authentication): Đảm bảo danh tính của các bên giao tiếp.
  • Không chối bỏ (Non-repudiation): Người gửi không thể phủ nhận đã gửi dữ liệu.

Các cơ chế phổ biến:

  • SSL/TLS: Bảo mật đường truyền.
  • Mã hóa (Encryption): Bảo vệ dữ liệu.
  • Chữ ký số (Digital Signature) và Chứng chỉ số (Certificate): Xác thực danh tính.

2. SSL/TLS là gì?

SSL (Secure Socket Layer) và TLS (Transport Layer Security) là giao thức mã hóa cho phép truyền dữ liệu an toàn giữa client và server qua Internet.
TLS là phiên bản nâng cấp và bảo mật hơn của SSL.

Khi bạn truy cập một website có tiền tố https://, nghĩa là kết nối đang sử dụng SSL/TLS.

3. Quy trình bắt tay TLS (TLS Handshake)

Khi client kết nối với server (qua HTTPS, SSL socket…), quy trình như sau:

Client ----> Server : ClientHello (phiên bản, mã hóa hỗ trợ)
Server ----> Client : ServerHello (chọn thuật toán mã hóa, gửi chứng chỉ)
Client ----> Server : Kiểm tra chứng chỉ, tạo khóa phiên (session key)
Server ----> Client : Xác nhận, bắt đầu truyền dữ liệu an toàn

Kết quả:
Hai bên chia sẻ khóa phiên bí mật để mã hóa/giải mã dữ liệu.

4. Mã hóa dữ liệu

4.1. Mã hóa đối xứng (Symmetric Encryption)

  • Dùng cùng một khóa cho mã hóa và giải mã.
  • Nhanh nhưng khó phân phối khóa an toàn.
  • Thuật toán phổ biến: AES, DES.

4.2. Mã hóa bất đối xứng (Asymmetric Encryption)

  • Dùng cặp khóa: khóa công khai (public key) và khóa bí mật (private key).
  • Dữ liệu mã hóa bằng khóa công khai chỉ giải mã được bằng khóa bí mật.
  • Thuật toán phổ biến: RSA, ECC.

4.3. Kết hợp

  • TLS thường dùng RSA/ECC để trao đổi khóa, rồi dùng AES để truyền dữ liệu (kết hợp hiệu suất và bảo mật).

5. Chứng chỉ số (Digital Certificate)

Chứng chỉ số chứa:

  • Tên tổ chức/cá nhân sở hữu.
  • Khóa công khai (public key).
  • Chữ ký số của tổ chức chứng thực (CA – Certificate Authority).

Khi trình duyệt hoặc ứng dụng kiểm tra chứng chỉ hợp lệ, nó tin rằng server là “thật”.

Ví dụ: Let’s Encrypt, DigiCert, GlobalSign…

6. Tạo kết nối SSL trong Java

a. Server SSL đơn giản

import javax.net.ssl.*;
import java.io.*;
import java.security.KeyStore;

public class SSLServer {
    public static void main(String[] args) throws Exception {
        // Tải keystore (chứa khóa bí mật và chứng chỉ)
        char[] password = "123456".toCharArray();
        KeyStore ks = KeyStore.getInstance("JKS");
        ks.load(new FileInputStream("serverkeystore.jks"), password);

        KeyManagerFactory kmf = KeyManagerFactory.getInstance("SunX509");
        kmf.init(ks, password);

        SSLContext sc = SSLContext.getInstance("TLS");
        sc.init(kmf.getKeyManagers(), null, null);

        SSLServerSocketFactory ssf = sc.getServerSocketFactory();
        SSLServerSocket serverSocket = (SSLServerSocket) ssf.createServerSocket(8443);

        System.out.println("SSL Server đang chạy trên cổng 8443...");

        while (true) {
            SSLSocket socket = (SSLSocket) serverSocket.accept();
            BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
            PrintWriter out = new PrintWriter(socket.getOutputStream(), true);

            String msg = in.readLine();
            System.out.println("Client gửi: " + msg);
            out.println("Server đã nhận qua SSL: " + msg);
        }
    }
}

b. Client SSL

import javax.net.ssl.*;
import java.io.*;

public class SSLClient {
    public static void main(String[] args) throws Exception {
        SSLSocketFactory ssf = (SSLSocketFactory) SSLSocketFactory.getDefault();
        SSLSocket socket = (SSLSocket) ssf.createSocket("localhost", 8443);

        BufferedReader in = new BufferedReader(new InputStreamReader(socket.getInputStream()));
        PrintWriter out = new PrintWriter(socket.getOutputStream(), true);

        out.println("Xin chào Server an toàn!");
        System.out.println("Phản hồi: " + in.readLine());

        socket.close();
    }
}

c. Tạo keystore (chứng chỉ thử nghiệm)

keytool -genkeypair -alias serverkey -keyalg RSA -keystore serverkeystore.jks -keysize 2048

7. Ví dụ mã hóa dữ liệu bằng AES

import javax.crypto.Cipher;
import javax.crypto.KeyGenerator;
import javax.crypto.SecretKey;
import java.util.Base64;

public class AESCrypto {
    public static void main(String[] args) throws Exception {
        KeyGenerator keyGen = KeyGenerator.getInstance("AES");
        keyGen.init(128);
        SecretKey key = keyGen.generateKey();

        String data = "Hello Secure World!";
        Cipher cipher = Cipher.getInstance("AES");
        cipher.init(Cipher.ENCRYPT_MODE, key);
        byte[] encrypted = cipher.doFinal(data.getBytes());

        System.out.println("Dữ liệu mã hóa: " + Base64.getEncoder().encodeToString(encrypted));

        cipher.init(Cipher.DECRYPT_MODE, key);
        byte[] decrypted = cipher.doFinal(encrypted);

        System.out.println("Giải mã: " + new String(decrypted));
    }
}

Kết quả:

Dữ liệu mã hóa: xwI4e2R+o1h...
Giải mã: Hello Secure World!

8. Các phương pháp bảo mật nâng cao

Phương pháp Mô tả
HMAC Mã xác thực thông điệp, chống giả mạo dữ liệu
JWT (JSON Web Token) Xác thực người dùng trong REST API
HTTPS (HTTP + TLS) Bảo mật toàn bộ giao tiếp HTTP
VPN / SSH Tunnel Tạo kênh truyền mã hóa an toàn
Firewall & IDS Bảo vệ mạng khỏi tấn công bên ngoài

9. Thực hành bảo mật REST API

Một REST API an toàn thường có:

  1. HTTPS – mã hóa toàn bộ dữ liệu.
  2. Token Authentication (JWT/OAuth2) – xác thực người dùng.
  3. Input Validation – tránh SQL Injection, XSS.
  4. Rate Limiting – giới hạn yêu cầu để chống DDoS.

Ví dụ tiêu đề HTTP:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6...

11. Kết luận

Bảo mật mạng là nền tảng của mọi ứng dụng hiện đại.
Việc áp dụng SSL/TLS, mã hóa dữ liệu, và các chuẩn xác thực an toàn giúp hệ thống:

  • Chống nghe lén và tấn công trung gian (MITM),
  • Bảo vệ thông tin người dùng,
  • Tăng độ tin cậy và uy tín hệ thống.
