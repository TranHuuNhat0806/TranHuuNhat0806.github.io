Bài 7 - Lập trình HTTP và REST API cơ bản

Đăng ngày 20/10/2025 lúc 00:00

1. Giới thiệu về HTTP và REST API

HTTP (HyperText Transfer Protocol) là giao thức truyền thông được sử dụng phổ biến nhất trên Internet, cho phép client và server trao đổi dữ liệu (văn bản, hình ảnh, JSON, v.v).

REST API (Representational State Transfer) là một phong cách thiết kế cho phép giao tiếp giữa các hệ thống thông qua giao thức HTTP, thường dùng trong các ứng dụng web và mobile.

REST hoạt động dựa trên tài nguyên (resource), được định danh bằng URL, và thao tác thông qua các phương thức HTTP như GET, POST, PUT, DELETE.

2. Kiến trúc RESTful

REST chia hệ thống thành hai phần:

  • Client: gửi yêu cầu (request) đến server.
  • Server: xử lý yêu cầu và trả về phản hồi (response).

Cấu trúc tổng quát:

Client  →  Request (HTTP)  →  Server  →  Response (JSON/XML)

Một REST API tuân theo các nguyên tắc:

  1. Stateless – Mỗi request độc lập, không lưu trạng thái giữa các lần gọi.
  2. Client-Server Separation – Client và server tách biệt rõ ràng.
  3. Uniform Interface – Dữ liệu và đường dẫn tài nguyên được thiết kế thống nhất.
  4. Use of HTTP Methods – Sử dụng đúng ngữ nghĩa của GET, POST, PUT, DELETE.
  5. Representation – Dữ liệu có thể ở nhiều định dạng (JSON, XML, v.v).

3. Cấu trúc HTTP Request & Response

3.1 HTTP Request

Một request HTTP bao gồm 3 phần:

<HTTP Method> <URL> <HTTP Version>
<Headers>
<Body>

Ví dụ:

POST /api/login HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "username": "admin",
  "password": "1234"
}

3.2 HTTP Response

Response cũng có cấu trúc tương tự:

<HTTP Version> <Status Code> <Message>
<Headers>
<Body>

Ví dụ:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "status": "success",
  "message": "Login successful!"
}

4. Các phương thức HTTP phổ biến

Phương thức Chức năng Ví dụ
GET Lấy dữ liệu /api/users
POST Tạo mới dữ liệu /api/users
PUT Cập nhật toàn bộ dữ liệu /api/users/1
PATCH Cập nhật một phần dữ liệu /api/users/1
DELETE Xóa dữ liệu /api/users/1

5. Mã trạng thái HTTP (Status Code)

Mã Nhóm Ý nghĩa
200 Thành công OK
201 Thành công Đã tạo (Created)
400 Lỗi client Yêu cầu sai (Bad Request)
401 Lỗi xác thực Không có quyền truy cập (Unauthorized)
404 Lỗi client Không tìm thấy tài nguyên
500 Lỗi server Lỗi nội bộ máy chủ

6. Ví dụ minh họa REST API (Java)

a. Server REST đơn giản với Java (sử dụng com.sun.net.httpserver)

import com.sun.net.httpserver.*;
import java.io.*;
import java.net.InetSocketAddress;

public class SimpleHttpServer {
    public static void main(String[] args) throws IOException {
        HttpServer server = HttpServer.create(new InetSocketAddress(8080), 0);
        server.createContext("/api/hello", new HelloHandler());
        server.setExecutor(null);
        server.start();
        System.out.println("Server đang chạy tại http://localhost:8080/");
    }

    static class HelloHandler implements HttpHandler {
        @Override
        public void handle(HttpExchange exchange) throws IOException {
            if ("GET".equals(exchange.getRequestMethod())) {
                String response = "{"message":"Xin chào từ REST API!"}";
                exchange.getResponseHeaders().add("Content-Type", "application/json");
                exchange.sendResponseHeaders(200, response.length());
                try (OutputStream os = exchange.getResponseBody()) {
                    os.write(response.getBytes());
                }
            } else {
                exchange.sendResponseHeaders(405, -1); // Method Not Allowed
            }
        }
    }
}

Cách chạy:

javac SimpleHttpServer.java
java SimpleHttpServer

Truy cập http://localhost:8080/api/hello để xem phản hồi JSON.

7. Gọi REST API từ Client (JavaScript)

fetch("http://localhost:8080/api/hello")
  .then(response => response.json())
  .then(data => {
    console.log("Phản hồi từ server:", data.message);
  })
  .catch(err => console.error("Lỗi:", err));

Khi chạy trên trình duyệt, console sẽ hiển thị:

Phản hồi từ server: Xin chào từ REST API!

8. Định dạng dữ liệu trong REST API

REST thường sử dụng:

  • JSON (JavaScript Object Notation) – dễ đọc, nhẹ, phổ biến nhất.
  • XML (Extensible Markup Language) – dùng trong hệ thống cũ.
  • Form Data / x-www-form-urlencoded – dùng trong HTML form.

Ví dụ JSON:

{
  "id": 1,
  "name": "Nguyen Van A",
  "email": "a@example.com"
}

9. Ưu điểm của REST API

  • Đơn giản, dễ triển khai, dễ hiểu.
  • Hoạt động tốt trên nền HTTP – không cần giao thức riêng.
  • Dễ tích hợp với web, mobile, IoT.
  • Hiệu suất cao, dễ mở rộng.

Nhược điểm:

  • Stateless – không lưu phiên (session).
  • Không phù hợp cho truyền tải dữ liệu phức tạp (so với gRPC, GraphQL).
  • Dễ sai sót nếu không tuân chuẩn RESTful.

10. Ứng dụng thực tế

  • Web Application: Giao diện web gọi REST API để lấy dữ liệu.
  • Mobile App: Ứng dụng Android/iOS dùng REST để đồng bộ với server.
  • Microservices: Các dịch vụ nhỏ giao tiếp qua REST.
  • IoT Devices: Thiết bị gửi dữ liệu cảm biến lên REST API server.

11. Kết luận

HTTP và REST API là nền tảng cho lập trình mạng hiện đại.
Việc hiểu rõ cách hoạt động của request/response, status code, và cấu trúc RESTful giúp lập trình viên phát triển các hệ thống web và mobile mạnh mẽ, dễ bảo trì, và tương thích cao.
