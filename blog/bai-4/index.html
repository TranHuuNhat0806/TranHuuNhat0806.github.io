Bài 4 - UDP Socket – Giao tiếp không kết nối (Connectionless Communication)

Đăng ngày 20/10/2025 lúc 00:00

1. Giới thiệu về UDP Socket

UDP (User Datagram Protocol) là giao thức truyền dữ liệu không kết nối (connectionless), cho phép gửi và nhận dữ liệu nhanh chóng mà không cần thiết lập hay duy trì phiên kết nối giữa client và server.

Trong lập trình mạng, UDP Socket được dùng khi ta cần tốc độ cao hơn độ tin cậy tuyệt đối – ví dụ như truyền video, âm thanh, hoặc dữ liệu game thời gian thực.

Đặc điểm chính:

  • Không cần thiết lập kết nối trước khi gửi dữ liệu.
  • Không đảm bảo thứ tự hoặc độ tin cậy của gói tin.
  • Dễ triển khai, tốc độ nhanh, độ trễ thấp.
  • Dữ liệu được gửi theo từng datagram độc lập.

Ví dụ:
Mỗi gói tin UDP chứa thông tin đích (IP:Port) và dữ liệu → gửi đi ngay, không chờ phản hồi xác nhận.

2. Cơ chế hoạt động của UDP

UDP hoạt động đơn giản hơn TCP vì không có giai đoạn handshake. Giao tiếp chỉ bao gồm việc gửi và nhận các datagram.

a. Gửi dữ liệu

  • Ứng dụng tạo một DatagramPacket chứa dữ liệu và địa chỉ đích.
  • Dữ liệu được gửi qua DatagramSocket.send().
  • Không cần chờ phản hồi, không xác nhận gói tin có đến nơi hay không.

b. Nhận dữ liệu

  • Ứng dụng lắng nghe qua DatagramSocket.receive().
  • Khi có datagram đến, dữ liệu được đọc vào bộ đệm (buffer).
  • Mỗi datagram được xử lý độc lập, không có trạng thái phiên.

c. So sánh TCP vs UDP

Đặc điểm TCP UDP
Kết nối Có kết nối (handshake) Không kết nối
Độ tin cậy Đảm bảo Không đảm bảo
Thứ tự dữ liệu Có Không
Tốc độ Chậm hơn Nhanh hơn
Kiểm soát lỗi Có Giới hạn
Ứng dụng phù hợp Truyền file, web, chat Game, video, streaming, IoT

3. Mô hình UDP Client - Server

Vì UDP không có kết nối, client và server hoạt động độc lập:

  • Server chỉ cần mở cổng để lắng nghe các gói tin đến.
  • Client gửi trực tiếp gói tin đến địa chỉ server.
    Không có bước xác nhận kết nối.

Quy trình tổng quát

Thành phần Vai trò Phương thức chính
Server Nhận dữ liệu DatagramSocket.receive()
Client Gửi dữ liệu DatagramSocket.send()
Cả hai Tạo gói tin DatagramPacket

4. Ví dụ lập trình UDP Socket với Java

File: UDPServer.java

import java.io.*;
import java.net.*;

public class UDPServer {
    public static void main(String[] args) throws IOException {
        DatagramSocket serverSocket = new DatagramSocket(5000);
        byte[] receiveBuffer = new byte[1024];
        System.out.println("Server đang chờ dữ liệu...");

        while (true) {
            DatagramPacket receivePacket = new DatagramPacket(receiveBuffer, receiveBuffer.length);
            serverSocket.receive(receivePacket);

            String message = new String(receivePacket.getData(), 0, receivePacket.getLength());
            System.out.println("Nhận từ client: " + message);

            String reply = "Server đã nhận: " + message;
            byte[] sendBuffer = reply.getBytes();

            InetAddress clientAddress = receivePacket.getAddress();
            int clientPort = receivePacket.getPort();

            DatagramPacket sendPacket = new DatagramPacket(sendBuffer, sendBuffer.length, clientAddress, clientPort);
            serverSocket.send(sendPacket);
        }
    }
}

File: UDPClient.java

import java.io.*;
import java.net.*;

public class UDPClient {
    public static void main(String[] args) throws IOException {
        DatagramSocket clientSocket = new DatagramSocket();
        InetAddress serverAddress = InetAddress.getByName("localhost");

        BufferedReader input = new BufferedReader(new InputStreamReader(System.in));
        System.out.print("Nhập tin nhắn gửi server: ");
        String message = input.readLine();

        byte[] sendBuffer = message.getBytes();
        DatagramPacket sendPacket = new DatagramPacket(sendBuffer, sendBuffer.length, serverAddress, 5000);
        clientSocket.send(sendPacket);

        byte[] receiveBuffer = new byte[1024];
        DatagramPacket receivePacket = new DatagramPacket(receiveBuffer, receiveBuffer.length);
        clientSocket.receive(receivePacket);

        String response = new String(receivePacket.getData(), 0, receivePacket.getLength());
        System.out.println("Phản hồi từ server: " + response);

        clientSocket.close();
    }
}

Cách chạy:

  1. Biên dịch: 
    javac UDPServer.java UDPClient.java
  2. Chạy server trước: 
    java UDPServer
  3. Sau đó chạy client: 
    java UDPClient

Đơn giản Không phải quản lý kết nối phức tạp

6. Ứng dụng thực tế của UDP Socket

  • Game online thời gian thực (FPS, MOBA, đối kháng) – cần tốc độ cao.
  • Truyền video, âm thanh trực tuyến (streaming) – mất vài gói tin không đáng kể.
  • Voice Chat (VoIP) – âm thanh liên tục, yêu cầu độ trễ thấp.
  • Broadcast trong mạng LAN – gửi thông báo đến nhiều máy cùng lúc.
  • Cảm biến IoT – gửi dữ liệu nhỏ, định kỳ về server.

7. Khi nào nên chọn UDP thay vì TCP?

Tình huống Giao thức khuyến nghị
Cần độ tin cậy tuyệt đối (file, ngân hàng, HTTP) TCP
Cần tốc độ, độ trễ thấp (game, streaming, IoT) UDP
Dữ liệu nhỏ, không quan trọng thứ tự UDP
Giao tiếp ngắn, không cần thiết lập phiên UDP

8. Kết luận

UDP Socket mang đến giải pháp truyền dữ liệu nhanh, hiệu quả cho các ứng dụng không yêu cầu độ tin cậy tuyệt đối.
Hiểu rõ đặc điểm connectionless, không xác nhận, và độc lập giữa các gói tin giúp lập trình viên thiết kế được các hệ thống mạng tối ưu về hiệu suất.

Trong khi TCP phù hợp với các ứng dụng đòi hỏi độ tin cậy cao, thì UDP lại là lựa chọn lý tưởng cho các ứng dụng thời gian thực, nơi tốc độ và phản hồi nhanh là yếu tố ưu tiên hàng đầu.
